8. Sélection de styles et types de média

Les feuilles de styles en cascades (CSS) sont un formalisme qui détermine l’apparence graphique de tout document conforme à XML : XHTML, SVG et autres. Ce formalisme se compose d’une part de propriétés (qui définissent un élément graphique : couleurs, dimensions, polices etc), d’autre part de sélecteurs (qui permettent de dire à quelles balises du document s’appliquent les dites propriétés, notamment celles possédant l’attribut class, voir le cours). Les sélecteurs relèvent de la programmation par règles, donc de l’informaticien, les propriétés relèvent plutôt des typographes et ne seront mentionnées ici que pour illustrer les problèmes programmatiques.

Ces styles sont le plus souvent placés dans des fichiers d’extension ".css" référencés à l’aide de l’attribut href des balises link dont l’attribut rel vaut stylesheet. Cette référence peut tenir compte des périphériques grâce à l’attribut optionnel media. Comme la valeur de l’attribut href peut être décidée dynamiquement, il est assez facile de tester sur un même écran les feuilles de styles prévues pour les différents périphériques, ou simplement pour des utilisateurs de goûts esthétiques divers. Le but de la présente séance est de voir comment construire des pages Web exploitant toutes ces fonctionnalités.


1 Les balises Link

Dorénavant nous allons produire des pages HTML utilisant des feuilles de styles, autrement dit des pages dont l’en-tête contient des balises link. Il faut donc redéfinir la fonction entête introduite à la séance L’environnement de PHP, car elle n’admet qu’un seul argument, le titre de la page. On souhaite maintenant que cette fonction admette un deuxième argument optionnel, qui sera un tableau dont chaque élément sera un sous-tableau des valeurs d’attributs de la balise link que l’on souhaite avoir dans l’en-tête. Ces sous-tableaux d’attributs auront le même format que ceux founis par SAX au preneur d’ouvrante. Dans le cas d’une feuille de styles, un sous-tableau sera par exemple :

array('rel'=>'stylesheet', 'href'=>'screen.css', 'media'=>'screen')

Ecrire une nouvelle définition de la fonction entête retournant un début de page HTML avec les balises link décrites par son nouvel argument.

Solution Télécharger entete.php

<?php
function entete($title, $links = array()) {

  $liens = '';
  foreach ($links as $link) {
    $atts = '';
    foreach ($link as $k => $v) $atts .= " $k='$v'";
    $liens .= "<link$atts />\n";
  }
  return
    "<!DOCTYPE html PUBLIC '-//W3C//DTD XHTML Basic 1.1//EN'
        'http://www.w3.org/TR/xhtml-basic/xhtml-basic11.dtd'>\n" .
    "<html xmlns='http://www.w3.org/1999/xhtml' lang='fr'>\n" .
    "<head>\n" .
    "<meta http-equiv='Content-Type' content='text/html;charset=utf-8' />\n" .
    "<title>" . 
    $title .
    "</title>\n" .
    $liens .
    "</head>\n";
}
?>

2 Sécuriser les choix

Afin d’expérimenter facilement les différentes présentations, il est pratique d’écrire une page XHTML sous la forme d’un script PHP qui accepte dans la Query-String de son appel un nom de feuille de styles à placer dans l’attribut href d’une balise link. Attention cependant : il est tentant pour un pirate d’essayer de détourner l’utilisation de cette Query-String.

On commencera donc par écrire une fonction get_securise, prenant en argument une chaîne, un tableau et une chaîne optionnelle égale à vide par défaut. Si le premier argument est un index défini du tableau donné en 2e argument, et si cet élément du tableau est une chaîne dépourvue de caractères dangereux, alors cet élément est retourné en valeur, sinon on retourne le 3e argument. L’intention est évidemment d’appliquer cette fonction sur le tableau $_GET (et aussi $_COOKIE comme on le verra).

Solution Télécharger get_securise.php

<?php
define('RE_SECU', '/^[^<>"\']+$/');

function get_securise($index, $t, $defaut='')
{
  if (isset($t[$index]) AND preg_match(RE_SECU, $t[$index]))
    return $t[$index];
  return $defaut;
}
?>

3 Changer de style

Les valeurs les plus courantes de l’attribut media sont screen (ordinateur), print (imprimante) et handheld (téléphone). Une bonne méthodologie de développement consiste à écrire autant de feuilles de styles, et donc insérer autant de balises link dans le document XHTML, plus une pour les styles communs à tous (attribut media égal à all ou absent). On se donne 5 feuilles de styles, objet des questions suivantes, nommées screen.css, print.css, handheld.css, all.css et all2.css, dont on souhaite voir les effets sur la table constitutant le fichier edt.html fourni en annexe. Pour ce faire, nous allons produire une page XHTML dont le corps contiendra cette table précédée d’une liste ul de deux items :

  • une balise a dont l’attribut href est une Query-String de 4 paramètres s, h, p, a de valeur respective screen.css, handheld.css, print.css, et all2.css ;
  • une balise a dont l’attribut href est une Query-String réduit aux 3 premiers paramètres de celle ci-dessus, leurs valeurs étant les mêmes à une permutation circulaire près.

L’entête de cette page comportera 4 balises link :

  1. les 3 premières auront comme attribut media respectif screen, handheld et print, et comme attribut href la valeur du paramètre de Query-string s, h, p respectivement, sous réserve de son acceptation par la fonction écrite précédemment, autrement on prendra les valeurs par défaut screen.css, handheld.css et print.css ;
  2. la quatrième n’aura pas d’attribut media et son attribut href vaudra :
    1. la valeur du paramètre a si elle est acceptable ;
    2. sinon la valeur du cookie homonyme s’il est présent et acceptable ;
    3. sinon la valeur all.css.

Dans le cas 2.1, il faudra de plus positionner le cookie espéré par le cas 2.2.

Ecrire un script PHP réalisant ces spécifications, en appliquant include sur l’annexe edt.html.

Solution Télécharger styles-table.php

<?php
include 'get_securise.php';
$s = get_securise('screen', $_GET, 'screen.css');
$h = get_securise('handheld', $_GET, 'handheld.css');
$p = get_securise('print', $_GET, 'print.css');
$a = get_securise('all', $_GET);
if ($a) {
  setcookie('all', $a);
} else {
  $a = get_securise('all', $_COOKIE, 'all.css');
}
include "entete.php";
echo entete("EDT Screen: $s, Handheld $h, Print $p",
    array(
     array('rel'=>'stylesheet', 'href'=>$a),
     array('rel'=>'stylesheet', 'href'=>$p, 'media'=>'print'),
     array('rel'=>'stylesheet', 'href'=>$h, 'media'=>'handheld'),
     array('rel'=>'stylesheet', 'href'=>$s, 'media'=>'screen')));
echo "<body>\n";
echo "<ul>";
$qs = "?screen=$h&amp;handheld=$p&amp;print=$s";
echo "<li><a href='$qs'>Rotation des périphériques</a></li>\n";
$qs = "?screen=$s&amp;handheld=$h&amp;print=$p&amp;all=" .
  (($a == 'all.css') ? 'all2.css' : 'all.css');
echo "<li><a href='$qs'>Permutation des styles</a></li>\n";
echo "</ul>\n";
include 'edt.html';
echo "</body></html>\n";
?>

4 Caractéristiques générales de la table

Le fichier edt.html fourni en annexe montre un extrait de la page d’accueil du site : il s’agit de la table XHTML Basic affichant les horaires et lieux des groupes de travaux dirigés. Plusieurs balises de ce document comportent un attribut class. Faites la nomenclature de leur valeur : elles référencent des styles qu’il faudra définir dans les quatre feuilles prévues à la question précédente.

Ecrire la feuille de style all.css, qui ne contient que les aspects invariants dans les 3 présentations, savoir :

  • la table occupe 100% de la largeur de la fenêtre ;
  • dans toutes les cellules, les textes sont centrés (propriété text-align) ;
  • la légende de la table à un fond de couleur grise (#ccc), et la graisse de sa police est en bold ;
  • la colonne indiquant le numéro de groupe utilise le style italique de la police ;
  • la colonne indiquant les intervenants utilise les petites capitales de la police ;
  • lorsqu’une balise td a pour tout contenu une balise ul, celle-ci n’a besoin ni de marge (propriété margin), ni de cadrage (propriété padding), et ses items, très courts ici, n’ont pas besoin de marqueur en début de ligne (propriété list-style-type).

Il faudra utiliser parfois les pseudo-classes first-child et last-child, ainsi que le signe de descendance immédiate ">".

Solution Télécharger all.css 
.calendrier-salle { text-align: center; width: 100%;}
.calendrier-salle caption { font-weight: bold; background: #ccc}
.calendrier-salle td+td+td[rowspan] {font-style: italic;} /* Groupe */
.calendrier-salle td+td+td+td[rowspan] {
 font-variant: small-caps; /* Les noms en petite capitales, c'est classe */
 font-style: normal; /* neutraliser la regle ci-dessus */
}
/* Si un TD est reduit a un UL, la presentation std est une perte de place */
.calendrier-salle td > ul:first-child:last-child {list-style-type: none; padding: 0; margin:0}

5 Des coûts et des couleurs

La table XHTML du texte fourni en annexe comporte 5 colonnes, de largeurs respectives 12%, 35%, 25%, 10% et 18%. Il suffit de les indiquer pour les colonnes de la première ligne, donc les balises th pour qu’elles s’appliquent aux suivantes. Par ailleurs, chaque balise td possède une bordure (propriété border) d’un pixel en ligne pleine (1px solid suivi au besoin d’une couleur). La couleur de cette bordure diffère selon les groupes, et est la même que la couleur du texte (propriété color) dans une colonne ; il s’agit à chaque fois d’une couleur foncée, tandis que le fond (propriété background) est une couleur claire dans la même gamme. Les couples (couleurs claire, couleur foncée) des groupes sont respectivement #EDF3FE et #5DA7C5 (Bleu), #EBE9FF et #766CF6 (Bleu pastel), #FEC et #FA9A00 (Orange), #FFEDED et #F00 (Rouge). Utiliser la valeur de l’attribut class des balises tr pour appliquer ces colorations.

D’autres occurrences de l’attribut class figurent dans des balises a sans aucun contenu. Afin de disposer d’une zone de clic, la classe doit comporter une propriété background référençant une image grâce à une valeur de la forme url(adresse) où adresse est une URL absolue ou relative (si le fichier de l’image est situé au même endroit que la feuille de styles), suivi du mot no-repeat si l’on ne souhaite pas que l’image se répète. Les images des deux premières colonnes se nomment agenda-24.gif et calendrier-24.gif respectivement. Leur taille est supposée s’inscrire dans un carré de côté 2 cadratins (unité em). On les placera à un cadratin du bord gauche de la colonne (propriété margin-left). La cinquième colonne comporte comme image un carré blanc de 3/4 de cadratin de côté, nommé puce-blanche.gif. Si l’on souhaite que ce carré figure à gauche du texte de la colonne, la propriété display doit-elle valoir block, inline ou inline-block ? Même question pour les 2 images précédentes si l’on veut les superposer verticalement.

Solution Télécharger screen.css 
.calendrier-salle th { width: 12%;} /* Groupe */
.calendrier-salle th+th { width: 35%;} /* Intervenant */
.calendrier-salle th+th+th { width: 25%;} /* Salle */
.calendrier-salle th+th+th+th { width: 10%;} /* Jour */
.calendrier-salle th+th+th+th+th { width: 18%;} /* Horaire */
.calendrier-salle td { border: 1px solid;}

.couleur2 /* Bleu */ {
	background: #edf3fe; border-color: #5da7c5; color: #5da7c5;
} 

.couleur3 /* Bleu pastel */ { 
	background: #ebe9ff; border-color: #766cf6; color: #766CF6;
} 

.couleur4 /* Orange */ {
	background: #fec; border-color: #fa9a00; color: #fa9a00;
} 

.couleur5 /* Rouge */ {
	background: #ffeded; border-color: #f00; color: #f00;
} 

/* Pour forcer la superposition verticale, display:block est indispensable */
a.abonnement {
	height: 2em;
	width: 2em;
	margin-left: 1em;
	display: block;
	background: url(agenda-24.gif) no-repeat;
}
a.telechargement {
	height: 2em;
	width: 2em;
	margin-left: 1em;
	display: block;
	background: url(calendrier-24.gif) no-repeat;
}
/* Pour que le texte qui suit ne provoque pas un saut de ligne, pas de bloc. */
/* Cependant display:inline ferait croire que cette balise vide est superflue */
/* display:inline-block est la seule solution */

.modif {
	width: .75em;
	height: .75em;
	display: inline-block;
	background: url(puce-blanche.gif)  no-repeat;
}

6 Table téléphonée

Lorsque l’on doit afficher une table XHTML très large sur un téléphone, il faut abandonner la présentation par colonne, le défilement horizontal étant pénible. Cela s’obtient en forçant la propriété display des balises td à block. Cet affichage entraine plusieurs conséquences :

  • la ligne de th ne doit plus être affichée (propriété display à none) car il n’y a plus de colonne ;
  • une balise td occupe 100% de la largeur de l’écran et l’utilisation d’une bordure n’a plus de raison d’être.

Par ailleurs, si on considère que les balises a du fichier edt.html indiquent dans leur attribut href (qu’on a omis ici) des pages inexploitables par téléphone (et c’est bien le cas dans l’original : les téléphones refusent de télécharger des calendriers alors qu’ils savent les gérer !), ces deux colonnes n’ont plus besoin d’être affichées. Quant au bouton de modification, il conduit à un service inexploitable par téléphone, on se dispensera donc de l’afficher. Pour ce faire, il suffit de ne PAS définir dans la présente feuille de styles les classes mentionnées dans l’attribut class de ces balises a pour qu’elles n’occupent aucune place sur l’écran du téléphone. Les images apparaissant dans ces balises a n’ayant été spécifiées que dans la feuille de styles screen.css, on évite donc ainsi de les charger pour rien.

Ecrire la feuille de styles souhaitée, qui devra en outre indiquer les mêmes couleurs que la précédente pour les tracés et les fonds (on aurait pu écrire ces spécifications communes dans une 5e feuille de styles référéncée par une balise d’attribut media égal à "screen, handheld".

Solution Télécharger handheld.css 
.couleur2 /* Bleu */   { background: #edf3fe;  color: #5da7c5; } 
.couleur3 /* Bleu pastel */ { background: #ebe9ff; color: #766CF6; } 
.couleur4 /* Orange */ { background: #fec; color: #fa9a00; } 
.couleur5 /* Rouge */  { background: #ffeded; color: #f00; }
/* Neutraliser la premiere ligne et les 2 premieres colonnes */
.calendrier-salle th, .calendrier-salle td[rowspan] { display: none;}
/* Pour toutes les autres, supprimer la bordure et occuper toute la largeur */
.calendrier-salle td, .calendrier-salle td+td+td[rowspan]
 { display: block; width: 100%; border: 0 }

7 Table imprimée

L’impression sur papier d’une telle page amène d’autres problématiques. Les hyper-liens dans la table ne servant alors à rien, on les ignore en ne définissant pas les classes qui les rendaient utiles via des images (il faudrait aussi ignorer les liens de changement de feuilles, mais ils sont nécessaires pour les besoins de l’exercice). Et surtout on veut souvent n’imprimer qu’en noir sur fond blanc, qui sont les couleurs par défaut, mais ce qui déclenche une série de conséquences :

  • pour que la table reste compréhensible, il faut remettre autour de chaque cellule une bordure de 1 pixel en ligne pleine, mais pas pour les cellules sans attribut rowspan qui :
    • ont une ligne en pointillés en bas (border-bottom:1 px dashed) pour la première ligne décrivant un groupe (la séance de travaux dirigés) ;
    • ont une ligne en pointillé en haut (border-top:1 px dashed) pour la deuxième ligne (la séance de travaux sur machine).
  • d’autre part, les 2 premières colonnes étant vides, on souhaite qu’elles n’aient pas de bordure afin de réduire leur aspect à une marge à gauche.

Pour réaliser ces spécifications, il est important d’avoir en tête l’algorithme de calcul de priorité des CSS vu en cours, qui précise que si deux règles s’appliquent, la plus prioritaire est celle ayant le plus grand nombre d’attributs dans son sélecteur, et si cela ne les départage pas, celle ayant le plus grand nombre de balises.

Solution Télécharger print.css 
/* compenser la disparition des couleurs par une bordure fine */

.couleur2 td,
.couleur3 td,
.couleur4 td,
.couleur5 td,
.couleur2 td+td+td[rowspan], 
.couleur3 td+td+td[rowspan],
.couleur4 td+td+td[rowspan],
.couleur5 td+td+td[rowspan] { 
	border: 1px solid;
} 

.couleur2 td+td+td+td+td,
.couleur3 td+td+td+td+td,
.couleur4 td+td+td+td+td, 
.couleur5 td+td+td+td+td{ 
	border-bottom: 1px dashed
}

.couleur2+.couleur2 td,
.couleur3+.couleur3 td,
.couleur4+.couleur4 td,
.couleur5+.couleur5 td {
	border-top: 1px dashed ;
}

.couleur2 td[rowspan], 
.couleur3 td[rowspan], 
.couleur4 td[rowspan],
.couleur5 td[rowspan] { 
	border: 0; 
}

8 Un défaut dans les défauts

Afin de tester l’usage du cookie dans le script écrit précédemment, il reste à écrire la feuille de styles all2.css. Les valeurs par défaut des navigateurs sont pour la plupart acceptables, sauf celle de la balise caption qui paraît insuffisamment intégrée à la table. On définira donc cette feuille en y mettant cette seule règle :

  • la balise caption de la table considérée aura un couleur de fond égale à #888.
Solution Télécharger all2.css 
.calendrier-salle caption { background: #888}
